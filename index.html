Method

Literal tools, hidden tests, signed-off numbers.

PPAbench is agent-agnostic. A model or scaffold submits artifacts; the harness reruns them inside pinned task environments. Scores are derived from parsers over Verilator, cocotb, Yosys, OpenROAD, Magic, and netgen output.

Why PPAbench

The missing full-stack chip benchmark.

Existing public evals usually stop at Verilog generation, simulation, synthesis, or human-authored RTL-to-GDS flows. PPAbench keeps the model in the loop and grades the whole path.

No public benchmark grades the whole pipeline

VerilogEval and VeriGen stop at simulation. RTLLM goes to synthesis. ChipBench and ASIC-Agent-Bench stop at module level. ORFS CI reaches GDS, but assumes human-authored RTL.

PPAbench grades spec to signoff

Natural-language requirements become RTL, tests, synthesis, P&R, DRC, LVS, timing, and final signoff metrics on real PDKs. The headline score is Power, Performance, and Area versus golden.

Tasks

One directory, one locked evaluation contract.

Each task ships a public spec and private grading material. The agent sees the spec and toolchain; the harness mounts hidden tests, golden references, manifests, and locked flow files only at scoring time.

Anti-gaming

Defense in depth for a gameable metric.

PPA is easy to game if the benchmark is naive. PPAbench layers RTL, flow, task, and metadata defenses so a submission has to be correct and reproducible, not merely tuned to visible tests.

RTL / PPA level

  • Hard caps on area, power, gates, and macro count.
  • Locked synthesis and P&R configs from golden task files.
  • Standard-cell and macro allowlists with netlist audit.
  • Synthesis-attribute blacklist and stripped-attribute smoke tests.
  • Hidden grading testbench with boundary and mutation coverage.
  • Logical equivalence checks where task scale allows it.

Meta level

  • Spec obfuscation: no reference names or trade-name leakage.
  • Held-out paraphrases reported as the minimum score.
  • Visible public versus holdout score delta.
  • Canary GUIDs in task files for contamination probing.
  • Time-windowed eval by task release date.
  • Verified-tier scaffold disclosure and trajectory inspection.

Submit

Two paths: artifacts for public, Docker for holdout.

Public submissions package model-produced artifacts. Holdout submissions ship an agent image that runs server-side against tasks the agent never sees.

Path A В· public split

Run your agent, write one solution directory per task, grade locally for debugging, then package an MLPerf-style submission tree with predictions, logs, exact artifacts, trajectories, and compliance metadata. Maintainers re-grade from artifacts. 

ppabench grade tasks/gcd_sky130hd_minimal work/gcd/solution
ppabench package --runs runs/ --team your-team --split public

Path B В· holdout split

Submit an agent Docker image with a defined entrypoint. The image runs in an egress-restricted environment with read-only task input, writable solution output, and only the configured model API reachable. 

docker run --network=ppabench-egress-only \
  -v /task:/task:ro -v /solution:/solution \
  myorg/myagent:v1
