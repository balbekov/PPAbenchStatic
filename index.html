Benchmark Flow

From SocMate run to deterministic scorecard.

The prototype fixes the measurement boundary. SocMate can choose the internal architecture, but every submitted design must enter the same wrapper, harness, and metrics pipeline.

01 Agentic SocMate run

Headless SocMate receives the design requirements and generates RTL, integration collateral, tests, logs, and metrics artifacts.

→
02 Fixed MPW wrapper

The public top is always user_project_wrapper with fixed Wishbone, LA, IRQ, QSPI, and 38-bit GPIO assignments.

→
03 Hidden harness simulation

The harness drives Wishbone control, byte streams, optional flash, fixed seeds, timeouts, and hidden design-specific vectors.

→
04 PPA + KPI metrics

Open tool outputs become area, power, Fmax, gates, macros, SRAM use, protocol checks, and deterministic application KPIs.

→
05 Scorecard

A design passes only when the wrapper contract, hidden tests, PPA caps, and required KPIs all pass.

Method

Literal tools, hidden tests, signed-off numbers.

PPAbench is agent-agnostic, but the SocMate prototype gives it a concrete agentic path. The harness consumes artifacts and metrics from pinned task environments, then derives scores from deterministic simulation and implementation outputs.

  1. 1Prepare isolated SocMate workspace
  2. 2Run headless SocMate against requirements
  3. 3Check fixed wrapper and pinout contract
  4. 4Run hidden deterministic harness simulation
  5. 5Extract PPA and resource metrics
  6. 6Emit pass/fail KPI scorecard

Why This Matters

A full-stack benchmark with a real integration boundary.

The MPW wrapper turns generated RTL into a comparable SoC-level artifact. The hidden harness then tests behavior, protocol, and PPA without trusting the agent's own summary.

Clear public contract

Participants know the wrapper, Wishbone register bank, stream pins, QSPI reservation, clock assumptions, and PPA caps before they run the benchmark.

Hidden but deterministic grading

The harness keeps golden vectors and implementation checks out of the agent workspace, but every pass/fail and metric is produced by deterministic scripts.

Run The Prototype

One suite manifest, three prepared workspaces.

The prototype source lives in benchmarks/socmate_mpw/. Workspaces receive requirements, a headless launcher, a manifest, logs, and a metrics directory.

Defenses

Guardrails stay in the background.

Defenses remain part of the benchmark, but they support the main user workflow: understand the public contract, run the agent, and reproduce the deterministic scorecard.

Design-level checks

  • Fixed wrapper, pinout, clocks, handshakes, and QSPI rules.
  • Hard caps for area, power, clock, gates, macros, and SRAM.
  • Hidden deterministic tests for protocol and application KPIs.

Benchmark-level checks

  • Locked suite manifests and reproducible workspace manifests.
  • Private harness vectors and golden references at grading time.
  • Scaffold disclosure and trajectory inspection for verified tiers.

Submit

Two paths: artifacts for public, Docker for holdout.

Public submissions package model-produced artifacts. Holdout submissions ship an agent image that runs server-side against tasks the agent never sees.

Path A · public split

Run your agent, write one solution directory per task, grade locally for debugging, then package an MLPerf-style submission tree with predictions, logs, exact artifacts, trajectories, and compliance metadata. Maintainers re-grade from artifacts. 

ppabench grade tasks/gcd_sky130hd_minimal work/gcd/solution
ppabench package --runs runs/ --team your-team --split public

Path B · holdout split

Submit an agent Docker image with a defined entrypoint. The image runs in an egress-restricted environment with read-only task input, writable solution output, and only the configured model API reachable. 

docker run --network=ppabench-egress-only \
  -v /task:/task:ro -v /solution:/solution \
  myorg/myagent:v1
